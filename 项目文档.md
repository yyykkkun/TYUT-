## 1. 数据库配置与建表

### application.yml 核心配置

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mall?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&createDatabaseIfNotExist=true
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
  jpa:
    hibernate:
      ddl-auto: update        # 自动建表/更新表结构
  autoconfigure:
    exclude:                  # 未装Redis时禁用
      - ...RedisAutoConfiguration
      - ...RedisConnectionAutoConfiguration
      - ...RedisRepositoriesAutoConfiguration
      - ...RedisVectorStoreAutoConfiguration
```

### 建库命令

```sql
CREATE DATABASE IF NOT EXISTS mall DEFAULT CHARACTER SET utf8mb4;
```

JPA `ddl-auto: update` 会自动创建所有表，**无需手动执行 SQL**。

## 6. 核心业务流程

### 6.1 下单支付发货收货

```
买家：加购 → 结算(选积分/余额/礼品卡/优惠券) → 提交订单(pending_payment)
     → 支付(paid) → 等待发货 → 自动刷新看状态
卖家：管理后台 → 订单管理 → 点"发货"(shipping) → 自动发站内信给买家
买家：看到"已发货"+运单号 → 确认收货(completed) → 获得积分(1元=1分)
```

### 6.2 金额计算（抵扣顺序）

```
商品小计 → 优惠券 → 积分(100分=1元,上限20元) → 礼品卡(上限50元)
       → 会员折扣(95~98折) → 运费(满199免,自提免) → 余额(兜底) → 应付
```

### 6.3 积分规则

| 操作 | 积分变动 |
|------|----------|
| 下单使用 | -N分（100分=1元，上限2000分/单） |
| 确认收货 | +订单金额分（1元=1分） |
| 取消/退款 | 退回已扣积分 |
| 兑换优惠券 | -500分得10元券 |

### 6.4 订单状态流转

```
pending_payment → 付款 → paid → 卖家发货 → shipping → 确认收货 → completed
                  ↘ 取消 → cancelled           ↘ 退款审核 → 退回钱+积分
```

### 6.5 会员等级

| 等级 | 成长值 | 折扣 |
|------|--------|------|
| 普通会员 | 0–499 | 无 |
| 青铜会员 | 500–1999 | 无 |
| 白银会员 | 2000–4999 | 98折 |
| 黄金会员 | 5000–9999 | 97折 |
| 钻石会员 | ≥10000 | 95折 |

## 7. 实现方法汇总

### 后端技术栈
- Spring Boot 3.5.14
- Spring Security + JWT（无状态认证）
- Spring Data JPA（自动建表、分页查询）
- MySQL 8.x（主数据库）
- BCrypt（密码加密）

### 前端技术栈
- Vue 3 Composition API + TypeScript
- Vite 8 构建
- Pinia 状态管理
- Vue Router 4 路由
- Ant Design Vue 4 UI组件
- ECharts + vue-echarts（数据看板图表）
- GSAP（动画）

### 鉴权方式
- 方案A：同账号角色分离，`User.role` 字段区分 user/admin
- 管理员可访问 `/admin/*`，前端路由守卫 + 后端 `@PreAuthorize`
- 演示账号 `demo@mall.test / 123456` 预置为 admin

### Mock 降级策略
- `isMockSession()` 检测 token 是否为 mock 假token
- 真实后端模式（真JWT）：禁止降级mock，报真实错误
- Mock模式（假token）：后端不可用时走localStorage兜底
