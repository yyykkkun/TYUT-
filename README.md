# 电商平台前后端落地方案

本项目定位为综合电商平台，覆盖买家会员、商品浏览、促销活动、下单支付、物流履约、积分/优惠券、评价晒单、消息通知，以及后续推荐与数据分析能力。当前仓库包含前端 `FE` 与后端 `BE` 两部分，现阶段优先落地可运行、可演示、可继续扩展的买家端 MVP。

## 建设目标

1. 第一阶段实现标准电商闭环：注册登录、商品搜索/列表/详情、购物车、下单、模拟支付、订单查询、收货地址、评价。
2. 第二阶段补齐营销能力：优惠券、积分、礼品卡、团购、秒杀、组合套餐、会员等级权益。
3. 第三阶段建设运营与数据能力：浏览记录、商品推荐、PV/UV/IP 统计、访问路径、商品访问排行。
4. 第四阶段扩展第三方/个人店铺、客服、推送、二维码、小游戏等增强功能。

## 技术现状

- 前端：`Vue 3 + Vite + TypeScript + Pinia + Vue Router`。
- 后端：当前仅有目录占位，建议采用 `Spring Boot 3 + MyBatis-Plus/JPA + MySQL + Redis`，也可根据课程要求替换为同等 Web 技术栈。
- 数据库：建议以 MySQL 作为交易主库，Redis 承担会话、缓存、秒杀库存预扣、热点商品缓存。
- 文件存储：商品图片、晒单图片可先落本地或对象存储兼容服务，接口层保持统一资源 URL。

## MVP 用户流程

1. 游客进入首页，查看商品分类、热卖商品、特卖商品。
2. 用户通过关键词、分类、品牌、价格、销量、人气等条件筛选商品。
3. 用户进入商品详情，选择规格与数量，加入购物车或立即购买。
4. 未登录用户下单时进入登录/注册；匿名购买可使用手机号生成默认账户。
5. 用户选择收货地址、优惠券/积分抵扣、配送方式，提交订单。
6. 系统锁定库存并创建待支付订单。
7. 用户使用模拟支付完成付款，订单进入待发货/待收货。
8. 用户查看物流轨迹，确认收货后可评价晒单并获得积分。

## 前后端职责边界

### 前端负责

- 页面路由、状态管理、表单校验、列表筛选、分页排序、倒计时展示。
- 商品、购物车、订单、会员、营销等业务页面。
- 登录态持久化、接口错误提示、空状态/加载态/权限态。
- 面向演示的交互完整性，例如模拟支付、秒杀倒计时、物流轨迹展示。

详细方案见 [FE/README.md](./FE/README.md)。

### 后端负责

- 用户认证、权限控制、业务数据校验、交易一致性。
- 商品、库存、订单、支付、优惠、积分、物流等领域接口。
- Redis 缓存、库存扣减、推荐数据读取、访问日志采集。
- 数据库表结构、接口文档、统一错误码、测试数据。

详细方案见 [BE/README.md](./BE/README.md)。

## 推荐数据模型

### 用户域

- `user`：账号、手机号、邮箱、密码摘要、状态、注册来源。
- `member_profile`：昵称、头像、性别、生日、等级、成长值。
- `member_account`：余额、积分、礼品卡余额。
- `address`：收货人、手机号、省市区、详细地址、默认标记。

### 商品域

- `category`：多级分类。
- `brand`：品牌。
- `product`：商品 SPU，标题、描述、分类、品牌、状态。
- `sku`：商品 SKU，规格、价格、库存、销量、图片。
- `product_attr` / `sku_attr`：属性与规格。

### 交易域

- `cart_item`：购物车商品。
- `order`：订单主表，金额、优惠、收货信息、订单状态。
- `order_item`：订单商品明细。
- `payment`：支付方式、支付状态、流水号。
- `delivery`：物流公司、运单号、轨迹。

### 营销域

- `coupon` / `coupon_user`：优惠券模板与用户领取记录。
- `promotion`：团购、秒杀、套餐、满赠等活动主表。
- `promotion_item`：活动商品、活动库存、活动价。
- `gift_card` / `gift_card_txn`：礼品卡与流水。

### 内容与数据域

- `review` / `review_image`：评价与晒单图片。
- `browse_history`：用户商品浏览记录。
- `site_message`：站内信。
- `access_log`：访问日志，支持 PV/UV/IP 与会话分析。
- `recommendation`：离线推荐结果，按用户/商品维度缓存。

## 接口设计原则

- 统一前缀：`/api`。
- 统一响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

- 分页响应：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 0
}
```

- 登录认证：建议使用 JWT 或 Session，前端统一在请求拦截器中携带凭证。
- 幂等控制：下单、支付、确认收货等接口需要业务幂等号。
- 交易一致性：订单创建时校验价格、优惠、库存，后端以数据库数据为准。

## 迭代计划

### Sprint 1：基础框架与用户商品

- 前端搭建路由、布局、请求封装、Pinia Store。
- 后端搭建项目、数据库连接、统一响应、认证基础。
- 完成注册登录、商品分类、商品列表、商品详情。

### Sprint 2：购物车与订单闭环

- 完成购物车、收货地址、订单确认页、订单创建。
- 完成库存锁定、订单状态流转、模拟支付、订单列表/详情。
- 补充基础单元测试与接口联调数据。

### Sprint 3：会员与营销

- 完成积分、余额、会员等级、优惠券、礼品卡。
- 完成秒杀/团购的展示、倒计时、活动库存逻辑。
- 完成评价晒单、浏览记录。

### Sprint 4：数据与增强功能

- 完成访问日志采集、PV/UV/IP、商品访问排行。
- 完成离线推荐结果表与猜你喜欢接口。
- 补充站内信、活动通知、门店自提、在线客服入口。

## 验收标准

- 用户可从注册登录到商品购买、支付、确认收货、评价完成完整链路。
- 商品列表支持分类、关键词、品牌、价格、库存状态、排序与分页。
- 下单接口能正确处理库存不足、价格变更、优惠不可用、重复提交。
- 前端页面具备加载态、错误态、空状态，移动端与桌面端均可使用。
- 后端接口有基础测试数据和接口文档，核心交易接口有测试覆盖。
- 推荐与报表接口可在无大数据平台时先基于日志表和缓存结果演示。
